Events

Markers that start, interrupt, or end a process.

Start Event (None)

Triggers a process without a specific trigger.

Cue: “Process begins when the work is ready.”

Start Event (Message)

Starts the process when a message arrives.

Cue: “Inbox message kicks off the workflow.”

Start Event (Timer)

Starts on a schedule or time condition.

Cue: “Every Monday at 9 AM.”

Intermediate Event (Message)

Waits for or sends a message during the flow.

Cue: “Pause until response is received.”

Intermediate Event (Timer)

Delays or waits until a time condition.

Cue: “Wait 2 days before follow-up.”

Intermediate Event (Error)

Catches or throws a business error mid-process.

Cue: “Stop this path, handle error flow.”

End Event (None)

Completes the process or subprocess.

Cue: “Work is finished.”

End Event (Error)

Ends and throws an error to the parent scope.

Cue: “Escalate error out of subprocess.”

End Event (Terminate)

Stops all activity in the current scope.

Cue: “Cancel everything immediately.”

Gateways

Control how paths split and merge.

Exclusive Gateway (XOR)

Exactly one outgoing path is taken.

Cue: “Condition-driven: choose one.”

Inclusive Gateway (OR)

One or more paths can be taken.

Cue: “Take every condition that matches.”

Parallel Gateway (AND)

All outgoing paths are taken concurrently.

Cue: “Split into parallel work; join waits for all.”

Activities

Work performed in the process.

Task

A single unit of work (user, service, or manual).

Cue: “One step; no internal breakdown.”

Subprocess (Embedded)

Collapses a sequence of steps inside the process.

Cue: “Reusable chunk within the same pool.”

Call Activity

Invokes a reusable process or global task.

Cue: “Use when the same process is shared.”

Flows & Structure

How work and communication move.

Sequence Flow

Shows the order of activities within a pool.

Cue: “Solid arrow; internal process path.”

Message Flow

Represents communication between pools.

Cue: “Dashed arrow; only between participants.”

Pool

Represents a participant or organization.

Cue: “Top-level swimlane boundary.”

Lane

Subdivides a pool by role or team.

Cue: “Who does the work inside a pool.”

Modeling Rules

Core constraints that keep diagrams correct.

Message flows only between pools

Use message flows for cross-participant communication.

Cue: “Never connect lanes with message flow.”

Sequence flows stay within a pool

Sequence flows cannot cross pool boundaries.

Cue: “Inside one participant only.”

One start, one or more ends

Each process has at least one start and end event.

Cue: “Ensure every path reaches an end.”

Gateways split and rejoin

Match split and join types for clarity.

Cue: “XOR split + XOR merge; AND split + AND join.”
